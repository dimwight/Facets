Implements Eval dialect Form that can input and process named values.

Eval types

A Field is an Evaluable identifiable by its Label; InputField can accept input validated against contained Values.

Fields are defined within a Form, which reads and writes a Record in the following lifecycle:

  1. A Record is either read from encoded text or created empty for an arbitrary Form.
  2. A Form is created matching that specified by the Record and reads labelled Values from the Record into its InputFields.
  3. The Form is passed to a reporting or interactive Consumer.
  4. If an InputField is updated, the Form updates the Record to match.
  5. The Record is written to encoded text as required.

Java types

This package implements the Form dialect with the following types:

Field
Evaluable that can be referenced by its Label within a containing Form.
InputField
Field that evaluates to a fixed Value that can be set and validated against a contained Values.
CalculatedField
Field that evaluates its EvalCoded contents.
Record
EvalCoded container for labelled Values and the DateStamp of the Form that can process it.
Form
Context containing a DateStamp, a Record sharing its DateStamp, a Forms.Dialect and any number of Fields which it can update from the Record and evaluate (first validating InputFields against their contained Values).
FormConsumer
Container EvalCoded coding for objects that can report on or interact with a Form.
Forms
EvalTypes container for a set of Forms created from encoded text, retrievable by reference to DateStamps of decoded or default Records.
